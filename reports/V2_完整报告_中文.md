> 报告日期:2026-04-18
> 项目阶段:数据清洗 + 鼻部 ROI 模型训练完整迭代

---

## 一、项目概览

### 1.1 目标

给定术前侧脸照片,基于 4 种深度生成模型(AutoEncoder / Pix2Pix / CycleGAN / DDPM Diffusion)预测术后鼻部外观,为医美术前沟通提供可视化参考。

### 1.2 本轮(V2)核心改动

相对 V1 版本,本次迭代做了三件事:

1. **数据重构** — 从 2568 张原始照片经完整去重 + 视角分类 + 异常剔除,得到 **572 对训练用 pre/post 图像**
2. **mask 算法重写** — 从 MediaPipe(在侧脸漏检 40%+)切换到 **InsightFace 5 点关键点 + 沿鼻轴倾斜椭圆** 方案,漏检率降至 <1%
3. **训练策略调整** — 放弃全脸模型,仅训练 4 个鼻部专用模型(`*_nose`),并用新 mask 重新裁剪 `nose_roi_128/`,保证 train / inference 输入分布一致

## 三、Mask 算法(最终定版)

### 3.1 V1 vs V2 对比

| 方面 | V1(MediaPipe) | **V2(InsightFace + 倾斜椭圆)** |
|---|---|---|
| 关键点数 | 478 点(face mesh) | 5 点(眼×2 / 鼻 / 嘴角×2) |
| 侧脸鲁棒性 | 漏检率 ~40% | 漏检率 <1% |
| 椭圆方向 | 固定垂直 | **沿鼻轴自适应旋转** |
| 中心定位 | 多点均值(易漂移) | 眼中点 + 0.65 × 鼻轴向量(稳定) |
| 与真实鼻部贴合度 | 中 | 高 |

### 3.2 算法细节

```python
# 输入:InsightFace 5-point kps = [right_eye, left_eye, nose_tip, right_mouth, left_mouth]
eye_mid   = (right_eye + left_eye) / 2
nose_axis = nose_tip - eye_mid                            # 方向 + 长度
axis_len  = |nose_axis|

center    = eye_mid + 0.65 × nose_axis                   # 65% 下移,刚好到鼻尖上方
long_axis  = 0.75 × axis_len                              # 沿鼻轴方向
short_axis = 0.55 × axis_len                              # 垂直鼻轴方向
angle_deg = atan2(nose_axis.y, nose_axis.x) × 180/π      # 旋转角

# 绘制实心椭圆 → 25×25 高斯模糊(σ=9)软边 → 归一化到 [0,1]
```

## 四、Nose ROI 裁剪 · Phase 0a

### 4.1 新裁剪算法

基于 V2 mask 的非零区域紧致 bbox + 12% padding,再 pad 到正方形避免 resize 畸变:

```python
bbox = 找出 mask > 30 的像素最小包围盒
bbox_padded = bbox + 12% of max(bbox_w, bbox_h)
bbox_square = pad 到正方形(居中扩展较短的边)
nose_crop = aligned_image.crop(bbox_square).resize(128×128, LANCZOS)
```

### 4.2 效果

![nose_roi_new_50](../tmp/nose_roi_new_50.jpg)

572 × 2 = 1144 张 `nose_roi_128/` 图像,全部聚焦鼻部、朝向统一(右)、像素占比高。

## 五、Phase 0b · 推理代码同步

### 5.1 问题

`backend/inference.py` 调用 `ml.nose_roi.get_nose_roi_box()`,该函数原本走 MediaPipe → 与训练数据分布不一致 → 推理期 `_nose` 模型泛化能力打折。

### 5.2 改动

重写 `ml/nose_roi.py`:

- 用 **InsightFace buffalo_l** 替换 MediaPipe
- 算法与 Phase 0a 完全一致(5 点 → 倾斜椭圆 → bbox → padding)
- 实现 **fork-safe 单例**:`os.register_at_fork(after_in_child=...)` + PID 记录,防止 FastAPI 多进程下 ONNX runtime 句柄共享崩溃
- Fallback 三级:`det_size=320` → `det_size=640 + CLAHE` → 启发式 bbox

### 5.3 成果

训练输入 = 推理输入 = 同一算法产物,消除了 train-inference distribution mismatch。

## 七、测试集评估 · Phase 2

### 7.1 指标说明

- **SSIM**(结构相似性):0-1,越高越好,衡量亮度/对比度/结构三方面的相似度
- **LPIPS**(感知相似度):0-1,越低越好,基于 AlexNet feature 距离,更接近人眼判断
- **FID**(Fréchet Inception Distance):越低越好,衡量生成分布与真实分布的距离(dims=2048)

### 7.2 结果(test split,57 对)

| 模型 | SSIM ↑ | LPIPS ↓ | FID ↓ | 定性 |
|---|---|---|---|---|
| 🥇 **`autoencoder_nose`** | **0.7383** | 0.2536 | **140.84** | 鼻形清晰自然 |
| 🥈 `pix2pix_nose` | 0.6942 | **0.1917** | 180.79 | 细节更锐利 |
| 🥉 `cyclegan_nose` | 0.6424 | 0.1935 | 189.14 | 与 pix2pix 接近 |
| ❌ `diffusion_nose` | 0.1818 | 0.7349 | 406.76 | 噪声,未收敛 |

### 7.3 定性样本(AutoEncoder,8 对)

![autoencoder_nose eval](../artifacts/eval/autoencoder_nose/qualitative_grid.png)

左→右:术前 / 真实术后 / **生成术后**

**观察**:

### 7.4 模型选择建议

| 场景 | 推荐模型 | 原因 |

### 8.2 评测对照(仅供参考,任务不同)

V1 只评测了**全脸 4 个模型**,没有 `*_nose` 评测;V2 只评测**鼻部 4 个模型**。直接比较 SSIM 数值不公平(全脸包含大量易匹配的静态区域,鼻部只看最难的区域)。但可以看出:

| 指标 | V1 全脸最佳 | V2 鼻部最佳 | 说明 |
|---|---|---|---|
| SSIM | 0.87(autoencoder) | 0.74(autoencoder_nose) | 任务更难 |
| LPIPS | 0.15(pix2pix) | 0.19(pix2pix_nose) | 差距有限 |
| FID | 95.0(autoencoder) | 140.8(autoencoder_nose) | 分布压力大 |

**V2 的真正价值不在绝对分数,而在于**:
- 生成结果**只覆盖真正需要改变的区域**(鼻部),不会误改其他面部特征
- 推理期可以把生成的鼻部 **paste_back** 到原始全脸照,得到自然的整体效果
- 数据质量显著提升,后续迭代的天花板更高

## 十、已知限制与后续改进方向

### 10.1 数据规模瓶颈

572 样本对 GAN/Diffusion 生成任务偏少。建议:

- 数据增广(旋转 ±5°、弹性形变、Mixup)
- 跨机构数据合作扩至 5000+ 对
- 基于预训练 StyleGAN 做 rhinoplasty prior

### 10.2 Diffusion 模型未收敛

当前 DDPM 架构自 train from scratch。改进路径:

1. 用 Stable Diffusion img2img + ControlNet(nose landmark 引导)
2. 或仅训练 500 步时间步条件 U-Net,加到 200 epochs

### 10.3 评估指标单一

目前只有 SSIM/LPIPS/FID 三项客观指标。鼻整形临床评估的黄金标准是:

- **鼻整形医师主观评分**(视觉可信度 / 解剖合理性 / 术后自然度)
- **关键测量指标**(鼻背角、鼻唇角、鼻尖突出度 — 已在 `ml/landmarks.py` 中实现,可集成到评估流程)

建议下一轮加入 **manual_review 人工评分模板**(已在 `artifacts/eval/*_manual_review_template.csv` 预留)。

### 10.4 推理端集成

当前最佳模型 `autoencoder_nose` 已可直接挂到 `backend/serve.py` 的 `/api/predict` 接口:

```python
# 推理流程已就位:
# 1) 上传全脸 → 对齐到 256×256
# 2) ml.nose_roi.extract_nose_roi() → 128×128 nose crop
# 3) 走 autoencoder_nose 前向 → 128×128 生成鼻部
# 4) ml.nose_roi.paste_nose_back() → 贴回原始全脸,羽化边缘
# 5) 返回前端展示
```

建议把 `backend/serve.py` 默认模型切换到 `autoencoder_nose`,并在前端 Compare Tab 支持 4 个模型并排对比。
